# 开发规范

## Git 工作流

### 分支命名

- `main` - 主分支，稳定版本
- `develop` - 开发分支
- `feature/*` - 功能分支
- `bugfix/*` - 修复分支
- `hotfix/*` - 紧急修复

### 提交规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型：**

- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `style`: 格式调整
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例：**

```
feat(user): 添加用户登录功能

- 实现 JWT 认证
- 添加登录页面
- 编写单元测试

Closes #123
```

## 代码规范

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量/函数 | camelCase | `getUserInfo` |
| 类/组件 | PascalCase | `UserProfile` |
| 常量 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT` |
| 文件 | kebab-case | `user-profile.tsx` |

### 代码注释

```javascript
/**
 * 获取用户信息
 * @param {string} userId - 用户ID
 * @returns {Promise<User>} 用户信息
 */
async function getUserInfo(userId) {
  // 实现逻辑
}
```

## 代码审查

- 所有代码必须通过 PR 合并
- 至少需要 1 个 Reviewer 批准
- CI 检查必须通过
